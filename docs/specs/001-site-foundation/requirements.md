# Spec 001: Site Foundation

## Goal

Create the base structure for Abdul's personal portfolio website with a clean, single-page layout and reusable section components.

## User Stories

### US-001: Visitor can understand who Abdul is immediately

As a recruiter or hiring manager, I want to see Abdul's name, role, and focus area immediately so I can quickly understand his professional identity.

#### Acceptance Criteria

- Given the visitor opens the homepage
- When the page loads
- Then the hero section displays Abdul's full name
- And it displays a short title such as "IT Student | Linux & Systems Support | Home Lab Enthusiast"
- And it includes links to GitHub, LinkedIn, email, and resume

### US-002: Visitor can navigate to major sections

As a visitor, I want clear navigation so I can jump to About, Projects, Skills, Certifications, Education, and Contact.

#### Acceptance Criteria

- Navigation links scroll to the correct sections
- Navigation works on desktop and mobile
- Active/hover states are visible

### US-003: Website has a consistent layout

As a visitor, I want the site to feel cohesive so it looks professional and intentional.

#### Acceptance Criteria

- All sections use consistent spacing
- Cards use consistent border, padding, and typography
- The page does not feel cluttered
