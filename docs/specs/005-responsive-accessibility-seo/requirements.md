# Spec 005: Responsive, Accessibility, and SEO

## Goal

Make the portfolio usable on desktop, tablet, and mobile while following basic accessibility and SEO practices.

## User Stories

### US-010: Visitor can use the site on mobile

As a mobile visitor, I want the site to fit my screen without awkward scrolling or broken layout.

#### Acceptance Criteria

- Navbar works on mobile
- Cards stack vertically on small screens
- Text remains readable
- Buttons are easy to tap

### US-011: Visitor using assistive technology can navigate the site

As a screen reader user, I want headings, links, and buttons to be understandable.

#### Acceptance Criteria

- Page uses semantic HTML
- Images/icons have labels where needed
- Buttons and links have accessible names
- Color contrast is readable

### US-012: Portfolio appears professional in search/link previews

As Abdul, I want the site to have proper metadata so it looks good when shared.

#### Acceptance Criteria

- Page has title
- Page has meta description
- Open Graph tags are added
- Favicon is added
